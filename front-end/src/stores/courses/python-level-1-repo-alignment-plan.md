# Python Level 1 Repo Alignment Plan

## Problem

`python-level-1.ts` currently surfaces only about 115 of the 224 top-level folders in `instruction-material/Python-Level-1`.

This repo is large and legacy-heavy. It mixes:

- canonical early Python projects
- turtle drawing labs
- shape and art drills
- check-ins
- starter variants
- extra practice folders

Examples currently not surfaced in the course:

- `Animal-Face-Lab`
- `Arrow`
- `Asterisk`
- `Beetle-Artist-Python-with-Turtle`
- `Block-of-Cheese`
- `Block-of-Cheese-Starter`
- `Christmas-Tree`
- `Circle-of-Diamonds`
- `Clock`
- `Clover`

## Plan

1. Build an explicit taxonomy for the repo:
    - required Python 1 projects
    - optional turtle/art studio projects
    - check-ins and practice packs
    - duplicate or starter-only folders
2. Expand `python-level-1.ts` with an optional enrichment section instead of forcing every drawing exercise into the main progression.
3. Promote any truly important missing programming projects into the core course.
4. Normalize starter and duplicate folders so the course links one canonical item per concept.
5. Consider adding a repo README index that marks which Python 1 folders are canonical, optional, or legacy.

## Validation

- the public course no longer looks incomplete relative to the repo
- art drills and turtle experiments are either intentionally surfaced as optional material or clearly archived
