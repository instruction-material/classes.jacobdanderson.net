# Physics Curriculum Expansion Plan

## Setup and Tooling

Preferred IDE: `VS Code` or `PyCharm`, depending on whether the class uses more
Python-based modeling. Much of the course can also use browser tools such as
PhET, Desmos, and spreadsheets, so the coding environment should stay light.

Official links:

- [VS Code download](https://code.visualstudio.com/Download)
- [VS Code on macOS](https://code.visualstudio.com/docs/setup/mac)
- [VS Code on Windows](https://code.visualstudio.com/docs/setup/windows)
- [PyCharm installation guide](https://www.jetbrains.com/help/pycharm/installation-guide.html)
- [Python downloads](https://www.python.org/downloads/?m=landing)

macOS walkthrough:

1. Install Python 3.
2. Install either VS Code or PyCharm.
3. If using VS Code, add the `Python`, `Jupyter`, and `Pylance` extensions.
4. Create a simple course folder for simulation notebooks, graphing scripts, and exported lab data.
5. Keep browser access ready for Desmos and PhET, since many conceptual modules should stay interactive rather than code-heavy.

Windows walkthrough:

1. Install Python 3 and enable the PATH option during setup.
2. Install either VS Code or PyCharm.
3. If using VS Code, add `Python`, `Jupyter`, and `Pylance`.
4. Verify that `python --version` works in a terminal before starting any simulation or graphing module.
5. Keep Excel, Google Sheets, or another spreadsheet tool available for data tables and graph analysis.

Course-specific notes:

- No CMake setup is needed for this course.
- The tooling should support graphing, simulations, and lab writeups more than large software projects.

## Positioning

The current physics path should become more coherent as a two-stage sequence:

- `Intro to Physics`: concept formation, intuition, graphs, and accessible labs
- `Physics Level 2`: quantitative modeling, vectors, multi-step problem solving,
  and engineering-style applications

## Current Gaps to Address

- More structured experimental work
- More graph interpretation and data analysis
- Better bridge between conceptual and mathematical physics
- More visible real-world applications and engineering tie-ins
- More cumulative projects and simulation work

## Intro to Physics Expansion

### Additions

- Measurement, uncertainty, and significant figures
- Motion graphs as a full strand rather than a small side topic
- Forces and free-body diagrams with more repetition
- Energy as a unifying concept across mechanics
- Intro electricity and circuits as a gentle practical unit
- Waves, sound, and light as highly visual units

### Recommended Projects

- Video-based motion analysis with a phone camera
- Roller coaster energy storyboard
- Paper bridge or tower force analysis
- Basic circuit build and measurement log
- Sound wave or resonance demo

## Physics Level 2 Expansion

### Additions

- Stronger vector decomposition and coordinate-system reasoning
- Momentum and impulse with collisions
- Rotational motion and torque
- Universal gravitation and orbits
- More sophisticated electricity and circuits
- Thermodynamics and heat transfer
- Intro optics and modern physics bridge topics

### Recommended Projects

- Ballistics or projectile model comparison
- Elevator and scale force simulation
- Pulley or incline force analyzer
- Collision lab with momentum accounting
- Simple orbit simulator in Python

## Delivery Improvements

- Each module should have:
  - a concept lesson
  - a worked example set
  - a graph/data exercise
  - a mini lab or simulation
  - a reflection question

- Use PhET, Desmos, spreadsheets, and optionally Python notebooks for
  visualization
- Add more teacher-facing diagnostic checkpoints and misconception lists

## Suggested Long-Term Structure

- `Intro to Physics`
- `Physics Level 2`
- optional `Engineering Physics Lab`
- optional `Electricity and Circuits`
- optional `Applied Mechanics`
